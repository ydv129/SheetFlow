## 🔐 PHASE 1: Auth, Security & REST APIs
1. **Authentication:** Implement `Auth.js (NextAuth)` with Google OAuth. 
2. **Security First (Middleware & Valkey):** Write a strict `middleware.ts`. Integrate **Valkey** as the in-memory data store for strict API rate-limiting and session caching to prevent abuse.
3. **MongoDB Atlas (Metadata):** Create a Mongoose schema for `User` and `ProjectConfig`. 
   - *Strict Rule:* NEVER store actual spreadsheet row data in the database.
4. **Human-Designed REST APIs:** All endpoints in `/backend/api` must follow strict RESTful conventions (`GET`, `POST`, `PATCH`, `DELETE`) with standard status codes and JSON formats.

## 🔌 PHASE 2: The Excel Live-Link
1. **File System Access API:** Create a custom hook `useExcelLiveLink.ts`. Save the `FileSystemFileHandle` in `IndexedDB`.
2. **The Watcher:** Implement a polling interval (every 5 seconds) inside a Web Worker to check `file.lastModified`.
3. **Parsing:** Use `SheetJS (xlsx)` to convert the binary buffer to JSON instantly in the browser. 

---

## 🧠 PHASE 3: Local-First AI Engine
1. **WebLLM Integration:** Create a `useLocalAI.ts` hook. Initialize the WebGPU engine using `@mlc-ai/web-llm`.
2. **Tiered Strategy:** Default to `SmolLM2-360M`, trigger `Gemma-4-E2B` only for complex analysis.
3. **Zero-Cloud Guarantee:** Enforce that no fetch requests containing Excel JSON data are ever sent to the backend.
